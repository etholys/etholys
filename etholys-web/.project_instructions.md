# ETHOLYS Project Instructions

## CRITICAL: Text Encoding
- **NEVER** use `String.fromCharCode()` for special characters in JSX/TSX files
- **NEVER** use `\u00XX` unicode escape sequences in JSX text content
- **ALWAYS** use HTML entities for special chars in JSX: `&oacute;` `&aacute;` `&eacute;` `&iacute;` `&uacute;` `&ntilde;` `&iquest;` `&mdash;` `&middot;` `&Iacute;`
- For JS strings (alerts, confirms, CSV export, etc.): use `\u00f3` `\u00e1` etc. is OK since they won't render in HTML
- When using heredoc/bash to write files, HTML entities work perfectly

### Smart Import (Importación Inteligente) — v2 REFACTORED ✅
- **Route**: `/siep/import` — multi-step wizard UI
- **API endpoints**: `/api/import/analyze` (POST FormData → LLM extraction), `/api/import/confirm` (POST JSON → DB creation)
- **LLM**: Uses gpt-4.1 (non-streaming, temp 0.1, max 16000 tokens) via Abacus AI API
- **File support**: PDF (base64), DOCX (mammoth text extraction), XLSX (xlsx lib text extraction), CSV, TXT
- **Extraction v2** (structured):
  - `project.goal` — separated from specific objectives (stored in Project.goal field)
  - `objectives[]` — recursive tree with `indicators[]` array per node (each: {name, unit, baseline, target})
  - `sow[]` — sections with `items[]` array (lists, not free text) + fallback `content` string
  - `budgetLines[]` — with `unit` field (month, day, trip, etc.) + better personnel cost guidance
  - `activities[]` — extracted from LogFrame activities with startDate/endDate
  - `milestones[]`, `risks[]`, `confidence{}` — same as before
- **Import Confirm v2**:
  - Stores `project.goal` in Project model
  - Creates indicator children as separate Objective records (type='indicator') with individual name/unitOfMeasure/baseline/target
  - Creates Tasks from activities (with startDate/dueDate, creatorId from session)
  - SOW items joined as bullet list with `•` prefix
- **Preview UI tabs**: Proyecto (with goal field), SOW (bullet lists), Marco Lógico (indicators table per node), Presupuesto (unit column), Riesgos, Actividades (activities + milestones combined)
- **M&E Section**: Refactored to tabular layout — indicators shown as table (Indicador, Unidad, Base, Meta, Actual, Avance, Mediciones) with expandable measurement sub-rows. Supports both legacy `indicator` field and new `type='indicator'` children.
- **LogFrame Section**: Indicators displayed as separate table per node (not crammed into one line). Expand-all button. "Objetivo Específico" label (since goal is at project level).
- **Project Detail merged sections**: "Cronograma" + "Tareas" → single "Actividades" section
- **Entry points**: "Importar" button in /siep/projects header + sidebar nav under Proyectos group
- **Design decisions**:
  - No templates — accepts any donor format (USAID, BID, UE, FIDA, GIZ, PNUD, etc.)
  - Always human review before saving
  - Project enters as DRAFT
  - Budget auto-calculated from lines if project.budget is 0
  - Objectives tree created recursively with type inferred from depth

- **M&E shows ALL trackable objectives**: `indicatorObjs` filter broadened to include objective, outcome, output, activity, deliverable, indicator, goal, impact types (not just those with indicator text field). This ensures OPs, OCs etc. appear in M&E even without a dedicated indicator string.
